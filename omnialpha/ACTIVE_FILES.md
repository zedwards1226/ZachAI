## `bots/`
- `__init__.py`
- `kalshi_public.py` ‚Äî unauthenticated `/historical/*` + `/markets/{ticker}` puller (no API key needed)
- `kalshi_client.py` ‚Äî RSA-PSS authenticated REST client (live only; not yet active)
- `order_placer.py` ‚Äî paper-order writer; HARD LOCK via `assert_paper_mode()` at top of `place()` (live cutover requires deleting that line in a separate diff)
- `live_scanner.py` ‚Äî live universe scanner ‚Äî pulls Kalshi public markets, hands snapshots to strategies, places orders. Invalidates ctx_cache after each placement so concentration gate doesn't go stale intra-pass
- `trade_monitor.py` ‚Äî settles open paper trades + writes pnl_snapshots. Falls back to live `/markets/{ticker}` when local DB row is stale. Computes Kalshi-correct entry fee on win + loss
- `telegram_alerts.py` ‚Äî Jarvis-bot send-only with [OmniAlpha] prefix. Plain-English entry/exit/daily-summary/halt/startup messages with strike + close time, narrative EOD digest with per-strategy breakdown + open positions. Anti-spam: `notify_error` throttled 30 min per (where, exc-type, msg); `notify_startup` rate-limited to 1/hr via sidecar file
- `strategy_labels.py` ‚Äî codename ‚Üí plain-English lookup (`crypto_btc15m_midband` ‚Üí "BTC 15-minute middle-band") used by every Telegram-bound message in the bot. Single source of truth ‚Äî new strategies must add an entry here
- `risk_engine.py` ‚Äî 7-gate pre-trade filter: paper-mode, per-trade $ cap, liquidity, concentration (max parallel), **same-series-same-side bucket** (blocks stacking neighboring strikes of the same series + side ‚Äî added 2026-05-05 after BTCD T80999+T80899 stacked into one $48 correlated NO position and lost $49), drawdown/loss caps, cross-bot halt. All caps are % of LIVE capital so they compound
